# 博客访问测试指南

## 部署完成后检查清单

### 1. 基本访问测试

#### 访问主页
- 地址：`https://yourname.github.io`
- 检查：页面是否正常加载
- 预期：看到博客首页，包含文章列表

#### 检查文章页面
- 地址：`https://yourname.github.io/2025/08/18/我的第一篇博客/`
- 检查：文章内容是否正确显示
- 预期：看到完整的文章内容和格式

#### 检查关于页面
- 地址：`https://yourname.github.io/about/`
- 检查：页面是否存在且内容正确
- 预期：看到关于页面的内容

### 2. 功能测试

#### 导航菜单
- 点击导航栏的各个链接
- 确认能正常跳转到对应页面

#### 主题样式
- 检查页面布局是否正确
- 确认 CSS 样式是否正常加载
- 验证响应式设计（在不同设备上查看）

#### 链接功能
- 测试内部链接是否正常
- 检查外部链接是否在新标签页打开

### 3. 技术验证

#### HTML 结构
- 右键查看页面源代码
- 确认 HTML 结构完整
- 检查 meta 标签是否正确

#### 资源加载
- 打开浏览器开发者工具（F12）
- 查看 Console 是否有错误信息
- 检查 Network 标签页，确认所有资源都成功加载

#### SEO 基础
- 检查页面标题是否正确
- 确认 meta description 存在
- 验证文章的 canonical URL

### 4. 常见问题排查

#### 页面显示 404
- 确保 Actions 部署成功
- 检查 URL 是否正确
- 等待几分钟再刷新（DNS 缓存）

#### 样式加载失败
- 检查主题配置是否正确
- 确认 CSS 文件路径正确
- 清除浏览器缓存后重试

#### 图片无法显示
- 检查图片路径配置
- 确认图片文件已正确上传
- 验证图片格式是否支持

#### 链接无法访问
- 检查 _config.yml 中的 URL 设置
- 确认永久链接配置正确
- 验证文件名是否包含特殊字符

### 5. 性能检查

#### 加载速度
- 使用 Google PageSpeed Insights 测试
- 检查页面加载时间
- 优化图片和资源大小

#### 移动端适配
- 在手机浏览器中访问
- 检查响应式布局
- 验证触摸交互是否正常

### 6. 浏览器兼容性

#### 推荐浏览器
- Chrome（推荐）
- Firefox
- Safari
- Edge

#### 测试版本
- 最新版本的浏览器
- 移动端浏览器
- 不同操作系统

### 7. 完成测试后的优化

#### 如果一切正常 ✅
- 开始创作更多内容
- 考虑添加评论系统
- 配置统计分析工具

#### 如果发现问题 ❌
- 记录具体的错误信息
- 根据错误类型进行修复
- 重新部署并再次测试

## 测试结果记录

创建一个测试结果文档：
```
✅ 主页访问正常
✅ 文章页面显示正确
✅ 关于页面加载成功
✅ 导航菜单工作正常
✅ 主题样式正确显示
❌ 需要优化：加载速度
```

## 获取帮助

如果测试过程中遇到问题：
1. 查看浏览器控制台的错误信息
2. 检查 GitHub Actions 的部署日志
3. 在 Hexo 社区搜索类似问题
4. 提供详细的错误信息寻求帮助
