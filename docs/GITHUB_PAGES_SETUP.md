# GitHub Pages 配置指南

## 配置步骤

### 1. 进入仓库设置
1. 打开你的 GitHub 仓库
2. 点击右侧的 "Settings" 按钮
3. 在左侧菜单中找到 "Pages" 选项

### 2. 配置 Pages 设置
在 Pages 页面中：

#### Source（源码设置）
- 选择 **"GitHub Actions"** 作为部署源
- 这将自动使用我们之前创建的 Actions 工作流

#### Branch（分支设置）- 可选
如果你使用传统方式：
- Source 选择 "Deploy from a branch"
- Branch 选择 "main"
- 文件夹选择 "/ (root)"

### 3. 等待部署
- 保存设置后，GitHub 会自动开始部署
- 通常需要 1-5 分钟完成
- 页面会显示部署进度和状态

### 4. 访问博客
部署成功后，页面会显示你的博客地址：
- 格式：`https://yourname.github.io`
- 点击链接即可访问

## 验证部署

### 检查内容
1. 博客首页是否正常显示
2. 文章链接是否能正常访问
3. 关于页面是否存在
4. 主题样式是否正确加载

### 常见问题

#### 问题 1：404 错误
- 确保 Actions 部署成功
- 检查 Pages 设置是否正确
- 等待几分钟后刷新

#### 问题 2：样式缺失
- 检查主题配置是否正确
- 确认所有资源文件都已生成

#### 问题 3：链接无法访问
- 检查 _config.yml 中的 URL 设置
- 确认文件路径配置正确

## 自定义域名（可选）

### 1. 购买域名
推荐域名注册商：
- 阿里云
- 腾讯云
- Namecheap
- GoDaddy

### 2. 配置 DNS
在你的域名管理后台：
- 添加 CNAME 记录：`www` → `yourname.github.io`
- 或添加 A 记录：`@` → `185.199.108.153`

### 3. 在 GitHub 中设置域名
在 Pages 设置中：
- 在 "Custom domain" 中输入你的域名
- 启用 "Enforce HTTPS"

## 监控部署

### 部署状态
- ✅ 绿色：部署成功
- 🔄 蓝色：部署中
- ❌ 红色：部署失败

### 部署历史
在 Actions 页面可以查看：
- 每次部署的详细日志
- 部署时间和状态
- 错误信息（如果有）
